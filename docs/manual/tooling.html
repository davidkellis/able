12. Tooling & Next Steps

  • Spec-first: match spec/full_spec_v10.md; update it when behaviour changes. Track open items in spec/todo.md.
  • Shared fixtures: edit Able sources, export via bun run scripts/export-fixtures.ts, then run both interpreters (bun run scripts/run-fixtures.ts, go test ./pkg/interpreter).
  • Design alignment: capture decisions in design/ and keep PLAN.md / interpreter10/PLAN.md current.
  • Testing: ./run_all_tests.sh orchestrates TypeScript tests, fixture runs, and Go parity checks; use --typecheck-fixtures=strict before landing runtime changes.
  • Further reading: explore examples/*.able, interpreter tests (interpreter10/test/), and concurrency notes in design/.

Able emphasises clarity, expressive types, and cross-runtime parity. When semantics are unclear, exercise fixtures in both interpreters, compare against the Go implementation, and document findings for the next contributor.
