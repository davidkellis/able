1. Getting Started

Able programs live inside packages. Any package with a public fn main() -> void produces a runnable binary named after its package path. 

package helloworld

fn main() {
  print("Hello, Able!")
}

Blocks are expressions; the last expression is the value. Braces delimit blocks—indentation is cosmetic.

1.2 Toolchain Setup

# Go interpreters
cd v12/interpreters/go
go test ./...

Use Go ≥ 1.22.

1.3 Running Able Code

# Execute fixtures with the Go harness
go test ./pkg/interpreter -run ExecFixtures

# Export fixtures after edits
./v12/export_fixtures.sh

# Go fixture/typechecker suite
go test ./pkg/interpreter

Run both suites together via ./run_all_tests.sh --version=v12. Add ad-hoc .able files under a package or examples/ and execute them with the interpreter CLIs.

1.4 Project Layout

  • v12/: active workspace (interpreters, parser, stdlib, fixtures)
  • spec/: specs v1–v12; rely on v12 for behaviour
  • Archived workspace at the repo root: frozen reference (do not modify)
  • able.*: reserved stdlib namespace (auto-detected; override via lockfile or ABLE_MODULE_PATHS)
