1. Getting Started

Able programs live inside packages. Any package with a public fn main() -> void produces a runnable binary named after its package path. 

package helloworld

fn main() {
  print("Hello, Able!")
}

Blocks are expressions; the last expression is the value. Braces delimit blocks—indentation is cosmetic.

1.2 Toolchain Setup

# TypeScript interpreter
cd v11/interpreters/ts
bun install
bun test

# Go interpreter
cd ../go
go test ./...

Use Bun ≥ 1.2 and Go ≥ 1.22.

1.3 Running Able Code

# Execute fixtures with the TS interpreter
bun run scripts/run-fixtures.ts [--filter name]

# Export fixtures after edits
bun run scripts/export-fixtures.ts

# Go parity (runs fixtures & typechecks)
go test ./pkg/interpreter

Run both suites together via ./run_all_tests.sh --version=v11. Add ad-hoc .able files under a package or examples/ and execute them with the interpreter CLIs.

1.4 Project Layout

  • v11/: active workspace (interpreters, parser, stdlib, fixtures)
  • spec/: specs v1–v11; rely on v11 for behaviour
  • v10/: frozen reference (do not modify)
  • able.*: reserved stdlib namespace (discoverable via ABLE_STD_LIB)
