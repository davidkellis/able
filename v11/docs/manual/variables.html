2. Syntax & Expressions

Able uses braces for blocks, backtick strings with interpolation, and a small set of punctuation tokens. Blocks are expressions; the last expression produces the value.

2.1 Lexical Basics

  • Line comments start with #. There are no block comments.
  • Identifiers use ASCII letters/digits/_ (not starting with a digit).
  • Statements end at newline or ;. Trailing commas are allowed in lists/params.

2.2 Literals

  • Numbers: 123, 1_000, 3.14, 0xff, 0b1010.
  • Booleans: true, false; Nil: nil; void represents “no value”.
  • Characters: 'a', '\\n'.
  • Strings: backticks with UTF-8 data and interpolation `Hello ${name}`.
  • Arrays: [1, 2, 3]. Structs: Point { x: 1, y: 2 }.

2.3 Strings & Interpolation

Backtick strings interpolate any expression with ${expr}. Escape a backtick as `\\` and the interpolation opener as ${"{"} . Length APIs expose bytes/chars/graphemes (see stdlib).

2.4 Blocks as Expressions

total_label := do {
  total := compute()
  `Total: ${total}`
}

A block with no final expression yields void (truthy).

2.5 Operators & Topics

  • Declarations use :=; reassignment uses =.
  • Range: a..b inclusive, a...b exclusive.
  • Safe navigation: value?.field, value?.fn() short-circuit to nil.
  • Pipe forward |> uses the topic % on the RHS: value |> %.normalize(). If the RHS has no %, it must evaluate to a unary callable: value |> (@ + 1).
  • Placeholders @, @2 build lambdas; % is only valid inside a pipe RHS.

2.6 Truthiness

Only false, nil, and values of type Error are falsy. Everything else (including void) is truthy.
